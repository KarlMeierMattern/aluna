Aluna — Partner Sharing Flow

Sequence diagram · owner uploads snapshot · partner reads via opaque token · no partner login

Snapshot, not live sync

The partner sees what the owner last uploaded — cycle day, phase name, countdown, optional fertile window. Re-sync happens when the owner saves settings or daily logs (if sharing is enabled).

Minimal data on server

Neon never receives full symptom logs or notes. Only the scoped fields the owner chose to share are stored in share_snapshots.

Opaque token

The share link uses a random server-side token — not a JWT in the URL — so revocation is instant when partner_links.active is set to false.

Aluna — System Context

C4 Level 1 · cycle-tracking PWA with local-first health data

System boundary

Aluna is one product: a mobile-first PWA. Cycle logs and symptoms stay in the browser (IndexedDB). The server handles sign-in, push delivery, and optional partner snapshots only.

Local-first health data

Personal health records never leave the device unless the user exports JSON or uploads a minimal partner snapshot. Google OAuth proves identity — passwords are never stored.

Partner access

Partners open an opaque link (/share/[token]) and see a server-held snapshot — not live IndexedDB data and no partner login required.

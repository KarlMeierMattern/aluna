Aluna — Google Sign-In Flow

Sequence diagram · Auth.js JWT session · health data stays in IndexedDB

Session

Auth.js uses a JWT strategy — the session lives in an httpOnly cookie, not in Neon. No sessions table and no Google OAuth tokens are stored.

Neon writes

On first sign-in, Drizzle adapter creates a users row and an accounts row linking the Google account id. Returning users are matched by provider + account id.

After login

All cycle logs, symptoms, and notes go to IndexedDB only. The JWT cookie is used to gate server routes (push subscribe, partner share) — not to authorize IndexedDB reads.

Aluna — Next.js Application Components

C4 Level 3 · modules inside the Vercel-deployed Next.js app

Client boundary

App UI, IndexedDB, cycle engine, and the service worker all run in the browser. Health data flows only between UI and IndexedDB — never through API routes unless the user exports or uploads a partner snapshot.

Server boundary

Auth.js and API routes run as Vercel serverless functions. Drizzle is the only module that talks to Neon. Middleware (middleware.ts) guards authenticated routes.

Per-user IndexedDB

After sign-in, local storage is namespaced by Auth.js user.id so a shared device does not mix two users' cycle data.
